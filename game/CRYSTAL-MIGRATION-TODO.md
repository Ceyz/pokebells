## Architectural follow-ups (NOT in the original 11 steps)

### [WRAM-1] ✅ Banked WRAM snapshot — attestation v1.1 (DONE 2026-04-22)

Landed variant **C** (multi-window, same 8 KB total):
- `readRamSnapshot` now reads bank 0 (`0xC000..0xCFFF`, 4 KB) + bank 1
  (`0xD000..0xDFFF` with SVBK forced to 1 via the supplied `writeByte`, 4 KB).
  SVBK is restored to its pre-snapshot value after the read so gameplay isn't
  affected.
- `computeCaptureAttestation` feeds the SVBK byte that was active during the
  read into the SHA-256 so verifiers can confirm the snapshot came from bank 1.
- New capture-record field `svbk_at_capture` (expected = 1 under v1.1).
- `ATTESTATION_SCHEME` bumped to
  `sha256:block_hash+wram8k_bank0_bank1+svbk+signed_wallet+session_sequence:v1.1`.
  The v1 string is kept as `ATTESTATION_SCHEME_V1` for indexer legacy paths.
- Shell passes `state.gb.writeByte` into `buildCaptureProvenance` so the
  bank switch happens under real gameplay.
- Two new unit tests cover the bank-switch + restore behavior.

### [HP-1] ✅ 16-bit enemy HP reads (DONE 2026-04-22)

`shell.js updateTelemetry` now calls `state.gb.readWord` for
`enemyHpCurrent` / `enemyHpMax`. `RAM_ADDRS.enemyHpCurrent` = `0xD216`
(word start, big-endian), `RAM_ADDRS.enemyHpMax` = `0xD218`. Boss HP no
longer wraps at 256.

### [EVOLVE-1] Evolution = op:"evolve" append, NOT burn + re-mint

Decision (2026-04-22): evolution is an append-only inscription referencing the
original capture, NOT a new capture with the old one burned. Rationale:

- Inscriptions are immutable; the NFT identity IS the inscription id. Burning
  and re-minting breaks trading history, provenance, and marketplace listings.
- Crystal evolution = same individual, changed species. Natural append.
- Max 2 evolves per mon in Gen 2 (Chikorita → Bayleef → Meganium) =
  ~0.002 BEL total in inscription fees.

Indexer aggregates:
```
current_species = capture.species_id; for evolve in evolves(ordered): if evolve.from == current: current = evolve.to
```

Evolve inscription body:
```json
{
  "p": "pokebells",
  "op": "evolve",
  "schema_version": "1.4",
  "ref": "<capture_inscription_id>",
  "ref_attestation": "<capture.attestation>",
  "from_species_id": 155,
  "to_species_id": 156,
  "evolved_at_block": 123456,
  "evolve_reason": "level_14",   // or "stone:FIRE_STONE" / "trade" / "happiness_max"
  "level_at_evolve": 14,
  "new_moves_learned": [52],
  "signed_in_wallet": "tb1p...",
  "signature": "<wallet_sign(sha256(canonical_content))>"
}
```

Indexer checks:
- `ref` resolves to a registered capture
- `signature` valid for `capture.signed_in_wallet`
- `from_species_id` matches current state (prevents double-evolve)
- `(from, to)` is a legal Crystal evolution path

For in-game detection: shell.js should watch wPartyMon{N}.species. When it
changes outside battle (evolution animation completes), prompt "Evolution
detected — sign + inscribe?".

### [PC-1] Crystal PC box writer

`gen2-pc-storage.mjs` exports the constants, but `syncOwnedCollectionToPcBoxes`
returns `{ incomplete: true, reason: 'Crystal PC box writer not implemented' }`.
Consequence: the "owned captures mirror inside the in-game PC" feature is
disabled on Crystal until someone ports the writer. The hub's Trainer tab is
the source of truth either way — the PC mirror has always been a nice-to-have.

Port plan (order matters):
1. Resolve TODO_CRYSTAL_SAVE offsets at the top of `gen2-pc-storage.mjs` from
   `pokecrystal.sym` + `ram/sram.asm`. Boxes 1-7 live in SRAM bank 2, 8-14 in
   bank 3, main game data in bank 1.
2. Port `serializeBoxMon` to Crystal's 32-byte BOXMON layout (see
   `constants/pokemon_data_constants.asm` MON_* + `BOXMON_STRUCT_LENGTH`).
3. Port `encodeGen1Text` to Crystal's charmap (`charmap.asm` — different from
   Gen 1).
4. Port `calcChecksum` to Crystal's 16-bit rolling sum over the main save
   block.
5. Port `writeBoxData` + `syncOwnedCollectionToPcBoxes` to build the target
   box record, write SRAM, and recompute both main-data and box-bank checksums.

## Quick commands

```bash
# 1. Regenerate species catalog from disasm:
node tools/generate-gen2-species.mjs \
  --pokecrystal-root Z:/pokecrystal \
  --out game/gen2-species.mjs

# 2. Regenerate 502 sprite PNGs:
node tools/import-pokecrystal-sprites.mjs \
  --pokecrystal-root Z:/pokecrystal \
  --out tools/sprites-source-gen2

# 3. Bundle the sprite pack (502 PNGs padded to 56x56 + manifest template):
node tools/generate-gen2-sprite-pack.mjs \
  --source tools/sprites-source-gen2 \
  --out tools/sprites-out-gen2

# 4. Run Gen 2 unit tests (21 tests + 31 auxiliary):
node --test game/capture-core.test.mjs game/gen2-species.test.mjs \
  game/wallet-adapter.test.mjs game/signin-verify.test.mjs \
  game/pbrp/session-key.test.mjs

# 5. Rebuild the ROM (from Z:/pokecrystal):
PATH="$PATH:Z:/tools/rgbds:Z:/tools/w64devkit-extract/w64devkit/bin" \
  make -C Z:/pokecrystal crystal
cp Z:/pokecrystal/pokecrystal.gbc Z:/PokeBells/game/pokecrystal.gbc

# 6. Re-chunk the ROM + regenerate all three manifest JSONs:
node game/build-rom-manifest.mjs --rom game/pokecrystal.gbc \
  --out game/ --network bells-testnet \
  --content-base-url https://bells-testnet-content.nintondo.io/content/

# 7. Re-extract RAM addresses whenever the ROM is rebuilt:
node tools/extract-crystal-ram-addrs.mjs --sym Z:/pokecrystal/pokecrystal.sym
```
