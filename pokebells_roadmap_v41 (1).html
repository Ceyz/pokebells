// catch rate — formule réaliste
Basée sur la formule Pokémon Gen 1 originale, adaptée pour la blockchain. La dépense de $POKEBALL est toujours consommée (réussie ou ratée) — c'est un sink économique intentionnel.
Formule de base :
catchChance = (catchRate × (maxHP×3 − currentHP×2)) / (maxHP×3) × ballBonus × statusBonus

Variables clés :
· catchRate : 3 (Mewtwo/Mew) → 255 (Magicarpe/Rattata) — lu depuis la ROM
· ballBonus : ×1 ($POKEBALL normal), ×1.5 ($SUPERBALL), ×2.55 ($MASTERBALL futur)
· statusBonus : ×1.5 si Sommeil/Gelé, ×1.2 si Paralysé/Brûlé/Empoisonné
· Seuil de succès : valeur aléatoire 0–255 < catchChance → capture réussie

Règle de burn $POKEBALL :
· Capture réussie → 1 $POKEBALL brûlé + mint NFT Pokémon
· Capture ratée → 1 $POKEBALL brûlé quand même (réalisme + sink économique)
· HP > 80% du max → avertissement UI "trop fort !" avant tentative

// boucle de jeu complète (vue joueur)
1. Tu joues à Pokémon Red/Blue dans le navigateur (100% on-chain)
2. Tu mints des $POKEBALL depuis ton wallet Nintondo (holders PokéBells uniquement, cooldown 360 blocs ≈ 6h)
3. Tu affaiblis un Pokémon sauvage → tu tentes la capture → 1 $POKEBALL brûlé (raté ou réussi)
4. Capture réussie → le Pokémon est minté comme NFT dans ton wallet (stats = lecture directe ROM)
5. Tu reçois un drop : $STARDUST (0–5 selon rareté catchRate) + $CANDY NFT (type du Pokémon capturé)
6. Tu peux staker tes $POKEBALL pour farmer du $STARDUST passif
7. Tu peux booster les IVs de ton Pokémon avec un $CANDY du bon type (brûlé, +1 IV max 15)
8. Tu peux faire évoluer ton Pokémon quand il atteint le bon niveau (on-chain, irréversible)
9. Tu combats les arènes → victoire = $BADGE NFT (non-transférable, lié à ton wallet)
10. Avec 2 Pokémons du bon egg group + 10 $STARDUST → tu peux breeder un œuf
11. Tu bats la Ligue Pokémon → seed on-chain assigne un légendaire random (Artikodin/Électhor/Sulfura/Mewtwo/Mew)
12. Tu as une seule chance de capturer ce légendaire — si capturé, league_cleared consommé définitivement
13. Tournoi trustless → mise ton meilleur Pokémon contre un autre joueur, le gagnant prend tout
14. Tu gères ton équipe active (6 slots) via le flag "active": true dans chaque JSON Pokémon
Écosystème de tokens conçu pour créer une économie circulaire auto-régulée : chaque ressource a un sink (endroit où elle est détruite) qui évite l'inflation indéfinie. Le catch rate réaliste (formule Gen 1) renforce le sink de $POKEBALL même sur des captures ratées.

// implémentation technique
$POKEBALL + $SUPERBALL + $STARDUST → CAT20 via @cat-protocol/cat-sdk. Le craft de $SUPERBALL utilise un minter contract qui vérifie les inputs de burn avant de minter. Le staking utilise le pattern "pledge minting" : lock $POKEBALL dans un minter UTXO → reward $STARDUST au déblocage.

$CANDY + $BADGE → CAT721. Chaque NFT encode son type dans les metadata CBOR. Le covenant de burn vérifie la compatibilité type. $BADGE non-transférable via covenant wallet-binding.

Timing : tokens en Phase 7 (OP_CAT requis). Version simplifiée côté-client (JSON sans covenant) dès Phase 4 pour tester la mécanique.

BellsChain timing : 1 bloc ≈ 1 minute. Cooldown mint $POKEBALL = 360 blocs ≈ 6h. Staking paliers : 100/500/1000 blocs ≈ 1h/5h/10h.
OP_CAT activé sur Bells mainnet le 2025-12-25 (Noël). Phase 8 (tokens CAT20/CAT721) n'est plus gatée. Le prérequis tech est levé — reste à forker @cat-protocol/cat-sdk pour BellsChain (Scrypt vs Bitcoin Script différents) puis déployer.
// règles de jeu par complexité
LEVEL-LOCK
Transfer seulement après niveau X. Covenant vérifie le champ "level" dans le JSON de l'UTXO.
OP_CSV + covenant level
COOLDOWN CAPTURE
Un wallet ne peut minter qu'un Pokémon toutes les 360 blocs (~6h). Évite le spam de capture et préserve la rareté.
OP_CSV — dispo sans OP_CAT
WALLET BINDING
Pokémon capturé lié à l'adresse minteuse. Si transféré → statut "boxed" jusqu'à activation explicite par le nouveau propriétaire.
flag "status" + covenant
ÉVOLUTION ON-CHAIN
Brûler l'ancien UTXO + preuve niveau requis pour minter l'évolution dans le même PSBT. Irréversible. IVs et "bred" flag préservés.
burn input + mint output + covenant espèce
BREEDING LIMITÉ
2 Pokémons même wallet + egg groups compatibles + 10 $STARDUST brûlés → NFT œuf. Covenant vérifie "bred: false" sur les deux parents.
3 inputs + covenant bred + egg_group + CAT20 burn
LIGUE ONE-SHOT
Victoire Ligue détectée via $D730 → covenant assigne un légendaire (seed hash bloc). league_cleared consommé à la première capture réussie. Irréversible.
$D730 flags + covenant one-shot + seed hash
FUSION DE STATS
Brûler 3 Pokémons même espèce (bred: false tous les 3) → minter un avec IVs max. Utilité principale des doublons d'espèce.
3 inputs + OP_CAT species check
TOURNOI TRUSTLESS
2 joueurs déposent un Pokémon dans multisig 2-of-2. Résultat signé par les deux parties débloque le gagnant. Time-lock sécurité si désaccord.
multisig 2-of-2 + OP_CSV time-lock
